# Frontend

No build step, no framework. Two static pages served by `express.static('public')`:

| Page | Purpose | Inline blocks |
|---|---|---|
| `public/index.html` | Authenticated kanban app | none — loads `/app.css` + `/app.js` |
| `public/login.html` | Sign in / sign up | inline `<style>` still; script extracted to `/login.js` |

`index.html` at the repo root is leftover and not served.

## CSP

```
default-src 'self'
script-src 'self'
script-src-attr 'none'
style-src 'self' 'unsafe-inline' https://fonts.googleapis.com
font-src 'self' https://fonts.gstatic.com
img-src 'self' data: https:
connect-src 'self'
frame-ancestors 'none'
base-uri 'self'
```

`style-src` keeps `'unsafe-inline'` because `login.html` still ships inline CSS. Everything else is strict.

## Event delegation (replaces inline `onclick`)

Because `script-src-attr 'none'` blocks all `onclick=`/`onchange=` attributes, the app uses a tiny dispatcher. Each interactive element gets `data-action="funcName"` and optionally `data-args='[...]'` (JSON-encoded args). One click handler at the document level looks up the function and calls it.

```html
<button data-action="addTask">Add</button>
<button data-action="switchBoard" data-args='[{"id":42,"name":"Work"}]'>Work</button>
```

The registry is built in `public/app.js`. Adding a new interactive element:

1. Use `data-action="someFunction"` instead of `onclick`.
2. Make sure `someFunction` is defined as a function declaration (hoisted) in `app.js`.
3. Register it in the `__actions` object near the bottom of `app.js`.

For elements inside modal containers that need to swallow clicks (so the overlay's close handler doesn't fire), use `data-action="stop"`.

## Account menu pattern (mobile-friendly header)

The right side of the header is `theme · bell · avatar`. The avatar is a single button that toggles `#account-dropdown`, containing:

- User identity (avatar + name + email)
- Search (with `⌘K` hint)
- Help
- Export backup
- Sign out (danger styling)

On mobile (≤ 768 px) the username text and chevron hide, leaving just the 36 px avatar circle. Closes on outside-click and when the notifications dropdown opens.

## Boot path (instant paint)

`public/app.js#init()`:

1. **Synchronously** paints from a `boot_cache_v1` localStorage entry: `{ userId, boardKey, categories, tasks }`. Cards appear before any `fetch` resolves.
2. Fires all six bootstrap requests in one `Promise.all` (`/api/user`, `/api/boards`, `/api/boards/memberships`, `/api/boards/members`, `/api/categories`, `/api/tasks`, `/api/notifications`).
3. When the network response arrives, repaints only if the data differs from the cache.
4. Cache is invalidated on any non-GET to `/api/tasks` or `/api/categories`, and on user-id mismatch.

## Keyboard shortcuts

| Key | Action |
|---|---|
| `⌘/Ctrl + K` | Open global search |
| `⌘/Ctrl + N` | Focus the new-task input |
| `?` | Open the help modal |
| `Esc` | Close any open modal / dropdown |

All routed through one `keydown` listener in `app.js`. Disabled while typing in `INPUT`/`TEXTAREA`/`SELECT`.

## ARIA

Icon-only header buttons (sidebar toggle, theme, bell, account) all carry `aria-label`. The account menu uses `aria-haspopup="menu"`, `aria-expanded`, `role="menu"` / `role="menuitem"`.

## Adding a new feature

1. Write the JS in `public/app.js` as a function declaration.
2. Add it to `__actions` near the bottom of the file.
3. Use `data-action="..."` in the HTML (or in template literals generated by JS).
4. Run `npm test` to make sure the CSP-related tests still pass.
